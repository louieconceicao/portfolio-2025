Senior UX Designer

B2B Billing Platform UX

Over 8 years, I designed the full spectrum of features for LogiSense’s billing platform—from simple modals to complex multi-step configuration screens. The platform supports enterprise customers and is featured in demo videos.

Role
Sole UX Designer
Timeline
8 Years
Platform
Web / B2B Enterprise
Deliverables
User Research, UI, Prototypes, Workflow Design

The Challenge

The legacy billing interface had grown organically over time and was difficult to navigate. When asked to redesign, I started fresh rather than building on the old UI. The goal was to create a scalable, modular system that could handle complex configuration workflows while reducing cognitive load for users.

My Approach

01 — Define New Mental Models

I mapped user tasks, roles, and expectations to create a clear mental model for accounts, subscriptions, and billing workflows.

Placeholder Image / User Journey Mapping

02 — Create Workflow Structures

I restructured complex multi-step workflows and backend screens for configuration and admin tasks, ensuring modularity and scalability.

Placeholder Image / Workflow Diagram

03 — Visual Relationships & Cognitive Load

I built visual relationships between objects to reduce cognitive load, simplify navigation, and clarify user actions across multi-step processes.

Placeholder Image / Configuration Screen

04 — Collaborate with Engineering

Screens were API-driven, adhering to backend structures. I worked with engineering to ensure designs were feasible while influencing API and database structure where necessary.

Placeholder Image / API-Driven Screen Example

05 — Leverage the Design System

Components were built using the Figma design system, ensuring consistency and reusability across the platform.

The Solution

  • Rebuilt interface for clarity, consistency, and usability
  • Modular, reusable components scaled across features and customer requests
  • High-fidelity prototypes guided Product, Engineering, and customer alignment
  • Screens adhered to API-first principles while influencing API and DB structure where needed
  • Established user personas and task flows shaping ongoing product development
Placeholder Image / Final UI Screens

The Outcome

The platform now has a consistent, modular UI that supports complex workflows without overwhelming users. New features and customer-requested enhancements can be designed and implemented efficiently using the shared component system.

High-fidelity prototypes and a clear design language improved collaboration with Product and Engineering, reducing misalignment and accelerating delivery. Over my tenure, this approach shaped the foundation for ongoing product growth.
